# EasyNovel — 分阶段开发计划

---

## 阶段一：MVP（最小可行产品）

**目标：能创建书籍、写下一章（全自动管线）、看结果**

| 任务 | 预估工期 | 产出 |
|------|----------|------|
| 项目脚手架搭建（Vite + React + Hono + TypeScript） | 1 天 | 可运行的前后端项目 |
| LLM 配置页 | 0.5 天 | 配置 Base URL / API Key / Model |
| 书籍 CRUD（创建/列表/删除） | 1 天 | 书籍管理 + 文件系统存储 |
| 故事圣经编辑器 | 1 天 | Markdown 编辑 + 保存 |
| 规划师 Agent (Planner) | 1.5 天 | 读取设定 -> 输出大纲 |
| 写手 Agent (Writer) | 2 天 | 大纲 -> 正文（去 AI 味规则） |
| 审计员 Agent (Auditor) | 1.5 天 | 6 维度审计 -> 输出报告 |
| 修订者 Agent (Reviser) | 1 天 | 自动修复关键问题 |
| 章节编辑器 + 审计面板 | 1.5 天 | 三栏写作页 UI |
| 一键写下一章流程串联 | 1 天 | Plan -> Write -> Audit -> Revise |
| 章节列表 + 状态管理 | 0.5 天 | 展示/切换章节 |
| **合计** | **~12 天** | |

### 阶段一交付物
- 用户可配置 LLM
- 创建一本书，填写故事设定
- 点击"写下一章" -> AI 自动完成规划/写稿/审计/修订
- 在网页上阅读和编辑章节
- 章节状态标记（草稿/待审/已通过/需修订）

## 阶段二：体验完善

**目标：提升写作质量、人工审阅流程、导出功能**

| 任务 | 预估工期 | 说明 |
|------|----------|------|
| 字数治理（目标字数 +/-20% 控制） | 0.5 天 | 写完后自动压缩/扩展 |
| 审计结果可视化优化 | 1 天 | 更好的维度展示 + 修订建议展示 |
| 批量操作（批量通过/修订） | 0.5 天 | 提高审阅效率 |
| 导出 TXT | 0.5 天 | 全文导出 |
| 导出 EPUB | 1 天 | 手机/Kindle 阅读格式 |
| 创作简报功能（建书时传脑洞文档） | 1 天 | 自动生成初始设定 |
| 题材模板库 | 1 天 | 内置 5 种题材模板 |
| 当前焦点控制 | 0.5 天 | 用户可指定近期写作方向 |
| **合计** | **~6 天** | |

## 阶段三：进阶特性

**目标：续写导入、自动提取、AI 味深度检测**

| 任务 | 预估工期 | 说明 |
|------|----------|------|
| 续写导入（导入已有章节） | 2 天 | 解析文本 -> 自动建立设定 |
| 自动设定提取 | 2 天 | 写完后自动提取角色/物品 |
| 主题切换（亮/暗） | 0.5 天 | Dark mode |
| 响应式适配 | 1 天 | 平板/手机基本可用 |
| 写作统计 | 0.5 天 | 仪表盘展示写作数据 |
| AI 味检测增强 | 1 天 | 独立检测 + 具体修改建议 |
| **合计** | **~7 天** | |

## 阶段四：打磨与优化

| 任务 | 说明 |
|------|------|
| 错误处理与重试机制 | LLM 调用失败自动重试 + 友好提示 |
| 写作进度实时展示 | 流式输出或分步 loading |
| Prompt 优化迭代 | 持续改进生成质量 |
| 文件锁 + 并发保护 | 防止同时写入导致数据损坏 |
| 测试覆盖 | 核心管线单元测试 |
| 用户反馈收集 | 使用中的问题跟踪 |

---

## 技术债务与注意事项

- 所有章节和设定以 **本地文件系统** 存储（JSON + Markdown），不依赖数据库
- LLM 调用使用 **Vercel AI SDK** 简化流式处理
- 前端使用 **Zustand** 管理状态，与文件系统保持同步
- 管线设计为 **串行执行**，每个 Agent 完成后触发下一个
- 保留 **人工干预入口**：任何步骤用户可中断并手动编辑
